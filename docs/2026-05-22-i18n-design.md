# Full UI Internationalization (i18n) Design

**Goal:** Replace all ~136 hardcoded user-facing strings with i18next `t()` calls, add 8 new languages (ja, ko, fr, de, es, pt, ru, it), and make the Rust tray menu respond to the selected UI language.

**Architecture:** Extend the existing i18next + react-i18next setup. Frontend uses `t()` keys as before. Rust tray menu reads `ui_language` from Tauri store and looks up labels from a built-in HashMap — no external i18n library needed.

**Tech Stack:** i18next, react-i18next, Tauri store, Rust HashMap

## 3. Rust Tray Menu i18n

### 3.1 Tray Labels HashMap

In `src-tauri/src/tray.rs`, add a function `get_tray_labels(lang: &str) -> TrayLabels`:

```rust
struct TrayLabels {
    show_window: &'static str,
    hide_window: &'static str,
    start_recording: &'static str,
    stop_recording: &'static str,
    settings: &'static str,
    history: &'static str,
    account: &'static str,
    about: &'static str,
    quit: &'static str,
}
```

A `match` statement maps language code to a `TrayLabels` struct with static `&str` values. Fallback to English for unrecognized languages.

### 3.2 Menu Build Flow

1. `build_tray_menu()` reads `ui_language` from Tauri store (default `"en"`)
2. Calls `get_tray_labels(&lang)` to get translated strings
3. Uses those strings as menu item labels

### 3.3 Language Switch Flow

1. User selects language in frontend (WelcomeStep or AboutPane)
2. Frontend calls `update_config({ ui_language: "ja" })` → saved to store
3. Frontend calls `invoke('refresh_tray_labels')` (new command)
4. Rust reads updated `ui_language` from store, rebuilds menu via `refresh_tray()`

### 3.4 New Command

In `commands/misc.rs`:

```rust
#[tauri::command]
fn refresh_tray_labels(app: tauri::AppHandle) -> Result<(), String> {
    refresh_tray(&app);
    Ok(())
}
```

Register in `lib.rs` invoke handler.

## 4. What NOT to Translate

- `STT_PROVIDERS` / `LLM_PROVIDERS` labels — brand names and proper nouns
- `LANGUAGES` / `TARGET_LANGUAGES` — language names are intrinsic
- `console.log` / `console.error` — developer-only
- Technical identifiers (config keys, provider IDs)
