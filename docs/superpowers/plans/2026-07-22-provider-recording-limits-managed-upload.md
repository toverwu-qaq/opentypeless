# Provider Recording Limits and Managed Upload Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Resolve Issue #81 by enforcing provider-aware recording limits in Rust and enable TalkMore Cloud recordings up to ten minutes without crossing Vercel Pro's 4.5 MB request boundary or degrading ordinary short-recording latency.

**Architecture:** Rust owns a versioned provider-capability registry, config migration, and an authoritative monotonic recording deadline. The frontend renders the resolved capability and user choice but cannot extend the Rust deadline. TalkMore first ships a version-2 managed-audio capability and strict WAV/Ogg-Opus container validator. The desktop then adds a statically bundled Opus encoder only to the managed Cloud path, retaining exact WAV behavior below 3.5 MB and conservative 30-second fallback when the server capability is absent or stale.

**Tech Stack:** Rust 2021, Tauri 2, CPAL, Tokio, `opusic-c 1.6.1`, `ogg 0.9.2`, React, TypeScript, Next.js, Vitest.

## Global Constraints

- Complete and deploy the TalkMore parser/capability route before enabling desktop capability version 2.
- Work on TalkMore in the isolated cloud-usage worktree/branch; preserve old desktop compatibility.
- Direct/BYOK and streaming provider audio paths must not be transcoded.
- Short managed recordings at or below 3,500,000 WAV bytes must send the same WAV bytes as today.
- Ogg/Opus work never runs in the CPAL callback or WebView and requires no end-user system codec.
- Rust is the deadline authority on Windows, macOS, and Linux; the WebView timer is display-only.
- Reaching a limit gracefully finalizes and submits recorded audio exactly once.
- Missing, stale, disabled, or incompatible managed capability metadata resolves to the historical 30-second Cloud fallback.
- Never send a multipart body above 4,200,000 bytes, an audio part above 4,000,000 bytes, or a recording above 600 seconds to TalkMore.

- [ ] **Step 2: Implement payload/timeout selection**

```rust
fn cloud_timeout(duration_seconds: u32, payload_bytes: usize) -> Duration {
    if duration_seconds <= 60 { return Duration::from_secs(60); }
    let seconds = 60 + (payload_bytes as u64).div_ceil(32_000);
    Duration::from_secs(seconds.clamp(60, 180))
}
```

For WAV, keep existing bytes. For long audio, finalize/validate Ogg and refuse any body that cannot stay below server caps. Preserve existing retry count and idempotency metadata.

- [ ] **Step 3: Handle encoder degradation explicitly**

Initialization failure narrows that recording to the WAV-safe 109-second deadline. Mid-recording failure below threshold still submits WAV. Above threshold allow one worker re-encode; otherwise return `managed_audio_encode_failed`. An early encoded cap gracefully submits a valid prefix once.
